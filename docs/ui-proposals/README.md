# K-Storm UI 重构提案

## 当前页面的主要问题

1. 信息层级偏平
   - 顶部、左侧模板、右侧结果区都使用相近的白底卡片，视觉重心不明确。
2. 工作流状态感不强
   - “模板填写 → 运行概览 → 讨论过程 → 报告导出 → 历史记录” 是一条很清晰的链路，但目前更像堆叠内容，而不是分阶段流程。
3. 长页面滚动导致认知切换频繁
   - 左侧是长表单，右侧又是多块纵向堆叠，用户在“配置、观察、导出、回看”之间来回跳。
4. 科研产品气质还不够鲜明
   - 现在更像基础管理台，缺少“研究工作台 / 多 Agent 实验台 / 学术分析仪表盘”的辨识度。

## 提案 A：Research Cockpit
关键词：稳健、专业、最易落地

### 页面配置
- 顶部：品牌 + 当前 Run 状态 + 快捷操作
- 左栏：固定模板区，分为「核心输入」「补充条件」「文档上传」三个折叠段
- 右区上半：运行概览 + 时间线
- 右区中部：讨论过程，采用 Agent 卡片网格
- 右区下半：结构化 IR / 最终报告 / 历史记录，使用 Tabs 切换

### 适合场景
- 保留现有交互心智
- 想在较低改动成本下显著提升观感
- 适合当前 MVP 直接升级

### 推荐视觉参数
- 主色：深绿 #123D33
- 强调色：青绿 #1D6D5A
- 背景：雾白 #F4F7F5
- 卡片圆角：12px
- 阴影：极轻，突出专业感而非消费级炫技

## 提案 B：Workflow Studio
关键词：流程感、聚焦、引导性强

### 页面配置
- 顶部：项目名 + Stepper
- 主体分三段式：
  1. Step 1 输入模板
  2. Step 2 观察多 Agent 讨论
  3. Step 3 阅读报告与导出
- 每一阶段占据主视图，其他阶段以侧边摘要方式存在
- 历史记录、模型设置进入抽屉或全屏弹层

### 适合场景
- 面向首次使用者
- 希望降低“页面太满”的压迫感
- 更强调产品流程，而非后台监控感

### 推荐视觉参数
- 主色：墨蓝 #18263A
- 强调色：蓝紫 #516BFF
- 辅助色：浅灰蓝 #EEF2F8
- 组件圆角：16px
- 大块留白 + 大字号阶段标题

## 提案 C：Agent Observatory
关键词：高级、实验台、最有辨识度

### 页面配置
- 左侧窄边栏：导航与全局状态
- 中央主舞台：当前阶段重点内容
- 右侧情报栏：Timeline、模型分配、导出、历史
- 讨论区使用“Agent 柱状卡片流”，每个 Agent 有独立色条和状态
- 数据指标、运行阶段、文档数、轮次等以小型仪表块展示

### 适合场景
- 产品想建立独特审美识别
- 强调这是一个多 Agent 研究分析台
- 适合后续继续扩展更多实验性能力

### 推荐视觉参数
- 主色：炭黑蓝 #101722
- 卡片：深灰蓝 #182231
- 强调色：青色 #39C6B4 / 金色 #F2B84B
- 文本：高对比浅灰
- 边框：1px 半透明亮边

## 我个人的推荐顺序
1. **提案 A**：最适合你当前项目，重构收益和落地成本平衡最好。
2. **提案 C**：如果你想让产品更像“研究实验平台”，这个方向最有气质。
3. **提案 B**：如果未来有更多非技术用户，或者要强调 onboarding，这套会更顺手。
